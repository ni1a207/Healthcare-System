# ERP Healthcare System (пет-проект)

---

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;**Данный репозиторий представляет собой комплексный проект по системному и бизнес-анализу на базе существующей микросервисной архитектуры. Проект выполнен с целью глубокого погружения в техническую реализацию распределенных систем и отработки навыков создания исчерпывающей технической документации.**

---
## Структура репозитория
* **Docs** — артефакты системного и бизнес-анализа:
  * As is — текущее состояние системы
    * API — интеграционные интерфейсы системы.
    * Diagrams — C4, SQD, BPMN.
    * Services — информация о сервисах.
    * **[Docs/As is/Healthcare.md](https://github.com/ni1a207/Healthcare-System/blob/main/Docs/As%20is/Healthcare.md)** - Техническая спецификация о системе (с включением всего содержимого из вспомогательных папок выше)**
  * To be — UI + ТЗ для разработки
* **Java** — Исходный код (Backend). Базовый проект: java-spring-microservices.

---

## Цель проекта
Развитие и демонстрация компетенций  System/Business Analyst в следующих областях:
* **Реверс-инжиниринг кода:** Чтение кода и анализ бизнес-логики на Java 21 / Spring Boot 3.


* **Проектирование архитектуры, интеграций, процессов и интерфейсов:** Визуализация микросервисного взаимодействия и паттернов интеграции. Построение комплексных диаграмм для описания процессов и структуры системы AS IS.


* **Документация в формате .md:** Описание межсервисного взаимодействия (REST, gRPC, Kafka) в виде технической спецификации, которая связывает бизнес-требования с программной реализацией в одном репозитории рядом с кодом.

---

## Инструменты
Для подготовки документации использованы:

* **Draw.io:** Проектирование архитектуры в нотации C4 (Container, Component).


* **PlantUML:** Создание Sequence-диаграмм для визуализации сценариев взаимодействия.


* **Camunda Modeler:** Моделирование бизнес-процессов в нотации BPMN 2.0.


* **Figma:** Проектирование UI/UX прототипов пользовательских интерфейсов.


* **Insomnia / Swagger (OpenAPI):** Проектирование и спецификация REST-контрактов в YAML формате.


* **Git / IntelliJ IDEA:** Работа с исходным кодом и версионированием - написание проекта.
