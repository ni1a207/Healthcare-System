# Healthcare-System — TO-BE Documentation

Данный раздел содержит проектную документацию целевого состояния системы **Healthcare-System**.

Документация описывает что необходимо изменить и добавить в текущую кодовую базу для получения работоспособного MVP: нормализованная модель данных, корректные архитектурные паттерны, консистентность между сервисами, новые сервисы и интеграции.

Каждый FR (Functional Requirement) описывает один бизнес-процесс и включает: AS-IS и TO-BE, BPMN диаграммы, C4 контейнерную диаграмму с изменениями, ERD, спецификации API, алгоритмы работы сервисов и критерии приёмки.

## Дорожная карта MVP

Порядок реализации определён зависимостями между FR и приоритетом бизнес-ценности.

**Этап 1 — Управление пациентами (core)**
```
FR-003 (Создание пациента)
    ↓
FR-004 (Просмотр карточки пациента)
    ↓
FR-005 (Редактирование пациента)
    ↓
FR-006 (Удаление пациента)
```

**Этап 2 — Расписание и записи**
```
FR-007 (Управление расписанием врачей)
    ↓
FR-008 (Запись пациента на приём — staff) ← требует FR-007
    ↓
FR-010 (Самозапись пациента)  ← требует FR-007 + FR-009
```

**Этап 3 — Личный кабинет и оплата**
```
FR-009 (Личный кабинет пациента)  ← требует FR-003
    ↓
FR-011 (Оплата после визита)  ← требует FR-008
```

## Процесс обновления документации

После успешной разработки, тестирования и деплоя каждого FR на прод документация в папке TO-BE считается реализованной и переносится в папку AS-IS.

При переносе обновляются следующие артефакты в AS-IS:

- **Сервисы** — описание сервисов с учётом новых компонентов и изменений
- **API** — спецификации эндпоинтов (новые, изменённые, удалённые)
- **DB** — ERD и словари таблиц для всех затронутых баз данных
- **Диаграммы** — C4 контейнерная и компонентная диаграммы системы, sequence диаграммы
- **Матрица трассировки** — связь бизнес-требований с реализованными FR и API

Соответствующий FR в таблице "Функциональные требования MVP" переводится в статус ![TO-BE](https://img.shields.io/badge/Done-green), файл FR-XXX.md остаётся в TO-BE как исторический артефакт проектирования.
